🔧 開發部署規範

Health Check 模組

  • 檔案放置於 /opt/health_check/
  • 由 phpipam-cron 容器執行
  • Volume mount：/opt/health_check:/health_check

DHCP Monitor Daemon

  • Log 路徑：/var/log/dhcp_monitor.log
  • 設定檔：/health_check/config/dhcp_servers.json

資料庫連線

  • Host: phpipam-mariadb (容器內) / localhost:3306 (主機)
  • Database: phpipam

📐 全域開發規範

專案命名規則

前綴 類型 說明
DEV- 開發專案 具備完整功能的開發專案，必須提供 API
MCP- MCP 專案 Model Context Protocol 整合專案
DOC- 文件專案 純文件或說明類專案

DEV 專案 API 規範

⚠️ 強制要求：所有 DEV- 前綴的專案必須提供 API 介面，以供後續 MCP 整合使用。
  • API 形式：Python 函數模組 / REST API / CLI 介面 (擇一或多種)
  • 目的：讓 MCP 專案可透過 API 調用功能，實現 AI 助理整合
  • 文件要求：需在 README.md 或獨立文件中說明 API 用法
  • 範例：
    • DEV-phpipam-mac-manager → 提供 CLI 子命令供 MCP 調用
    • DEV-phpipam-health-monitor → 提供 PHP 腳本可被 cron 調用
